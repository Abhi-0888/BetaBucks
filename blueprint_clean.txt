Section 1  ·  Introduction & Vision

The Real-Time Stock Market Simulator is a full-stack MERN web application that replicates the experience of live stock trading — using real market prices from the National Stock Exchange of India (NSE) — while removing the financial risk entirely. Users trade with virtual money (₹1,00,000 starting balance) against actual, live-ticking stock prices.

1.1  Project Vision Statement

"Build a platform where a complete beginner can experience buying TCS at ₹3,842 and selling it at ₹3,917, see their portfolio go green, and compete on a real leaderboard — all without risking a single rupee."

2.3  Microservices Described (From Document)

SERVICE NAME

FULL DESCRIPTION



User Interface (React.js)

Frontend where users interact. Views stock prices, executes trades, monitors portfolio. Receives real-time updates via Socket.io events in a reactive UI.



Web Server (Express.js)

Handles incoming HTTP requests from the UI. Manages routing and forwards to the API Gateway. Acts as bridge between frontend and all backend services.



API Gateway

Routes requests to the correct backend service. Performs JWT validation, input sanitisation, and response formatting. Ensures smooth, consistent communication.



Auth & Access Control Service

Manages user authentication and authorization. Verifies credentials with bcrypt, issues JWTs, assigns user/admin roles, protects all private routes.



Profile & Portfolio Management Service

Manages user profiles, virtual balances, and portfolio data. Stores holdings, average cost per stock, and account details. Enables investment tracking.



Simulated Trading & Order Service

Handles BUY and SELL orders with virtual currency. Fetches live NSE price at execution, validates balance/holdings, processes the trade, and updates portfolio atomically.



Real-Time Market Data Service

Fetches live NSE stock prices from Yahoo Finance every 15 seconds. Broadcasts price_update socket events to all connected clients. Ensures users see current data.



Leaderboard & Analytics Service

Tracks user performance. Calculates total portfolio value (cash + live holdings). Ranks users. Generates P&L charts, trade frequency stats, and sector allocation analytics.



Real-Time Trading Dashboard

Frontend module displaying live market data, user balance, portfolio overview, and trending stocks. The central hub for all trading activity.



Active Orders & Portfolio View

Shows current holdings, avg buy price, live price, unrealised P&L per stock, and overall portfolio allocation. Updates in real time via socket events.



Profit/Loss Analysis & Report

Calculates and visualises both realised and unrealised P&L. Shows daily P&L trends, win/loss rate, best/worst trades. Helps users evaluate strategy effectiveness.



User Trading History

Stores and displays every past trade. Filterable by date, stock, type (buy/sell). Supports CSV export. Used for performance analysis and review.



Trading Simulator Settings

Allows users to customise simulation parameters: max trade size, notifications, display preferences, reset balance option. Admin can control market hours.



User & Admin Accounts

Manages user identity, roles (user/admin), login credentials, and access permissions. Admin can view all users, reset balances, and manage the stock universe.

Section 9  ·  Trading Engine Logic & P&L Formulas

9.1  BUY Order — Step-by-Step Execution

STEP

ACTION

DETAIL



①

Receive Order

POST /api/trade/buy  {symbol: 'TCS.NS', quantity: 5}  — JWT required



②

Fetch Live Price

Call Yahoo Finance API → regularMarketPrice for TCS.NS → e.g. ₹3,842.50



③

Calculate Cost

totalCost = 5 × ₹3,842.50 = ₹19,212.50



④

Validate Balance

If user.virtualBalance < totalCost → throw 400: 'Insufficient balance'



⑤

Atomic Deduct

findOneAndUpdate({userId}, {$inc: {virtualBalance: -totalCost}}) — atomic operation



⑥

Create Transaction

type:'BUY', qty:5, pricePerShare:₹3842.50, totalAmount:₹19212.50, balanceBefore, balanceAfter



⑦

Upsert Portfolio

If holding exists: newAvgCost=(oldQty×oldAvg + qty×price)/(oldQty+qty) If new: create with avgCost=price, qty=5



⑧

Emit Socket

socket.to(userId).emit('trade_confirmed', {...}) + socket.to(userId).emit('portfolio_update')



⑨

Respond

200: {transaction, newBalance: ₹80,787.50, portfolioEntry}

9.2  SELL Order — Step-by-Step Execution

STEP

ACTION

DETAIL



①

Receive Order

POST /api/trade/sell  {symbol: 'TCS.NS', quantity: 5}  — JWT required



②

Check Holdings



③

Fetch Live Price



④

Calculate Proceeds

proceeds = 5 × ₹3,917.00 = ₹19,585.00



Calculate P&L

⑨

trade_confirmed with pnl:+₹372.50  |  portfolio_update  |  leaderboard may update

9.3  All P&L & Portfolio Value Formulas

METRIC

FORMULA

EXAMPLE



Weighted Average Cost

(oldQty × oldAvg + newQty × newPrice) ÷ (oldQty + newQty)

(5×₹3800 + 3×₹3900) ÷ 8 = ₹3837.50



Unrealised P&L (per stock)

(currentLivePrice − averageCostPrice) × quantityHeld

(₹3917 − ₹3837.50) × 8 = +₹636



Unrealised P&L %

(Unrealised P&L ÷ totalInvested) × 100

(636 ÷ 30700) × 100 = +2.07%



Realised P&L (on SELL)

(sellPrice − averageCostPrice) × quantitySold

(₹3917 − ₹3842.50) × 5 = +₹372.50



Total Portfolio Value

virtualBalance + Σ(qty × currentLivePrice) for all holdings

₹80,787 + ₹31,336 + ... = ₹1,04,250



Overall Return %

((totalPortfolioValue − 100000) ÷ 100000) × 100

((104250 − 100000) ÷ 100000) × 100 = +4.25%



Leaderboard Score

totalPortfolioValue (higher = better rank)

₹1,04,250 → Rank #3



Win Rate

(profitableSELLtrades ÷ totalSELLtrades) × 100

8 wins out of 12 sells = 66.7% win rate

Section 14  ·  Security Model

AREA

TECHNOLOGY

IMPLEMENTATION



Authentication

JWT (HS256)

7-day expiry. Stored in httpOnly cookie (prod) or localStorage (dev). verifyJWT middleware on all protected routes.



Password Storage

bcrypt.js

saltRounds: 12. Password never returned in any API response. Compare hash with bcrypt.compare() on login.



Rate Limiting

express-rate-limit

100 requests per 15 minutes per IP globally. 10 trade requests per minute per user. Returns 429 on breach.



HTTP Headers

helmet.js

Sets Content-Security-Policy, X-Frame-Options, X-XSS-Protection, Strict-Transport-Security, and more.



CORS Policy

cors middleware

Only FRONTEND_URL is whitelisted. All other origins blocked. Credentials allowed for cookie-based auth.



Input Validation

Joi schemas

All POST/PUT request bodies validated before reaching controllers. Rejects unknown fields. Returns 400 with detail.



Atomic Operations

MongoDB findOneAndUpdate

Balance deduction and portfolio update done atomically to prevent double-spend race conditions.



Socket Auth

Socket.io middleware

JWT verified on socket handshake. Socket joins personal userId room. Unauthenticated sockets cannot trade.



Secrets Management

.env + Render/Vercel

MONGO_URI, JWT_SECRET never in codebase. Validated at startup in config/env.js. Different values per environment.
